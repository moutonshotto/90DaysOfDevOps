Personalized DevOps Goals and Commitment Memorandum
    The aim of this document is first and foremost enforcing accountability. Clarity is important so that the goal is clear and visible at all times. 
    Accountability is needed to materialise my commitment every single day till I achieve my goal.

    My current level is that I am just a Java developer who works in developing and enhancing microservices. 
    But for me its not enough. I don't know much when it comes to shipping my application after development and handle production issues, 
    arising due to scalability or due to server crashes, and knowing ways to debug critical issues after the application goes live, or monitoring the existing application health.

    I have some of the goals in my mind, which I aim to achieve in 90 Days

    1. Build and Deploy a Production Grade Microservices Project.
       a. Design, containerize, and deploy a production-style Spring Boot microservices application using Docker, Kubernetes, Helm, and CI/CD pipelines on AWS/EKS.
       b. This project should be planned and implemented end-to-end while learning with the live lectures. 
       Outcome: One end-to-end deployable project demonstrating real DevOps workflow ownership.

    2. Possess Troubleshooting mindset and skills.
       a. Build familiarity with production troubleshooting tools such as:
          kubectl logs, observability, pipeline logs, docker logs & container inspection, Linux networking and process debugging commands.
       b. Prepare and maintain an excel sheet, which will have list of errors encountered, hotfix, permanent resolution, along with root cause analysis of the experienced issue.
       Outcome: Confidence in handling production incidents, reading logs/metrics, debugging failures, and performing end-to-end RCA independently.

    3. Develop Engineering Discipline and Consistency Systems.
       a. Maintain a daily commit in the github repository documenting topics learned, commands practiced, challenges faced, key learnings, pending doubts.
       b. At the end of every week, prepare a short weekly self review containing
          What was completed.
          What was not completed.
          Biggest blockers and future steps to deal with them.

    Weekends will be with occupied with live lectures and self study of the concepts learned.
    Weekdays will be on solving class assignments and deep diving into the weekend's lecture and concepts, hands-on practice.
    A daily goal will be set to cover certain topic of a particular technology, and until it is not achieved, there should be no closure. 
    So I am not going to impose any time limit, just a mindset, of achieving small tangible goals everyday.
